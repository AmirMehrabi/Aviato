# Admin Dashboard Rebuild Plan

## Goal
Redesign the admin dashboard from a cluttered, generic layout into a comprehensive, scannable operational view with charts, grouped categories, and clear visual hierarchy.

## New Layout Structure (4 sections, top to bottom)

```
┌──────────────────────────────────────────────────────────────┐
│  SECTION 1: STATUS STRIP — 5 compact mini-cards in a row     │
│  [Infrastructure] [VM Fleet] [Revenue] [Tickets] [Alerts]    │
├──────────────────────────────────────┬───────────────────────┤
│  SECTION 2: CHARTS ROW               │                       │
│  ┌────────────────────────────────┐  │  Server Health Bars   │
│  │ Revenue Trend (30-day chart)   │  │  (per-server bars)    │
│  └────────────────────────────────┘  │                       │
├──────────────────────────────────────┼───────────────────────┤
│  SECTION 3: OPERATIONS ROW           │                       │
│  ┌────────────────────────────────┐  │  Open Ticket Queue    │
│  │ Critical Alerts (card list)    │  │  (by priority)        │
│  └────────────────────────────────┘  │                       │
├──────────────────────────────────────┼───────────────────────┤
│  SECTION 4: FINANCIAL + ACTIVITY     │                       │
│  ┌────────────────────────────────┐  │  Recent Activity      │
│  │ Financial Overview (2x2 grid)  │  │  (timeline feed)      │
│  └────────────────────────────────┘  │                       │
└──────────────────────────────────────┴───────────────────────┘
```

## Files to Modify

### 1. `package.json` — Add chart.js dependency
```bash
npm install chart.js
```

### 2. `resources/js/app.js` — Register Chart.js globally
Add at top (after existing imports):
```js
import { Chart, registerables } from 'chart.js';
Chart.register(...registerables);
window.Chart = Chart;
```

### 3. `app/Http/Controllers/Admin/DashboardController.php` — Add new data queries + restructure return

**New queries to add:**
- Import `Ticket` model
- `$ticketsOpen` / `$ticketsPending` counts
- `$ticketByPriority` array (urgent/high/normal/low)
- `$recentTickets` — top 5 open/pending tickets ordered by priority
- `$revenueChartLabels` + `$revenueChartData` — 30-day revenue trend (daily sum of positive WalletTransactions)
- `$todayRevenue`, `$monthRevenue`, `$monthInvoiceSum`, `$monthInvoiceCount` — financial summary
- `$negativeWalletTotal` — sum of negative wallet balances

**Restructure `return view(...)` to pass:**
- `$statusStrip` — 5 mini-card data arrays (label, value, sub, tone, url)
- `$revenueChart` — labels + data for Chart.js
- `$serverHealth` — existing `$capacityRows`
- `$criticalAlerts` — `$attentionItems` filtered to red/amber only
- `$ticketStats` — open/pending counts + by_priority
- `$recentTickets` — top 5 tickets
- `$financial` — today/month revenue, invoices, negative wallets, pending payments
- `$recentActivity` — existing feed
- `$wallets` — WalletService instance (existing)
- `$health` — existing health array (kept for status strip)

**Keep all existing private methods unchanged** (`attentionItems()`, `capacityRows()`, `financialRisk()`, `recentActivity()`, `readinessScore()`, `percentage()`).

### 4. `resources/views/admin/dashboard.blade.php` — Full rewrite

**Section 1: Status Strip**
- 5 compact cards in `grid-cols-2 sm:grid-cols-3 lg:grid-cols-5`
- Each: colored dot (green/amber/red) + label + value + subtitle
- All clickable, linking to relevant index pages

**Section 2: Charts Row**
- `lg:grid-cols-[1fr_380px]`
- Left: Revenue trend area chart (Chart.js canvas, initialized via `x-init`)
  - Blue gradient fill, no points, smooth tension, RTL-aware
- Right: Server health horizontal bars (existing `$serverHealth` data)
  - Server name, detail text, progress bar, "manage servers" link

**Section 3: Operations Row**
- `lg:grid-cols-2`
- Left: Critical alerts as colored cards (NOT table) — filtered from attentionItems
  - Red/amber bg per tone, dot indicator, title, meta, action badge
- Right: Open ticket queue
  - Priority breakdown bar (colored segments)
  - Top 5 tickets with priority badge, subject, customer, time, status badge

**Section 4: Financial + Activity Row**
- `lg:grid-cols-2`
- Left: Financial overview — 2x2 grid of metric cards (today revenue, month revenue, wallet debt, month invoices)
- Right: Recent activity timeline (existing `$recentActivity` feed)

## Visual Style
- Keep existing patterns: `rounded-xl border border-slate-200 bg-white p-5 shadow-sm`
- Status dots: `size-3 rounded-full` with green/amber/red
- All cards have `transition hover:border-[#B8D6FF] hover:shadow-md` hover effects
- Empty states: `border-dashed border-slate-200 p-8 text-center`
- Chart.js colors: blue `#0069FF`, amber `amber-500`, red `red-500`

## Verification
1. `npm run build` — ensure Vite compiles with chart.js
2. `vendor/bin/pint` — ensure PHP formatting is clean
3. Load admin dashboard in browser — verify all 4 sections render, charts display, all links work
4. Test with empty data (no VMs, no tickets) — verify empty states render correctly
5. Test responsive layout — status strip should stack on mobile, charts/operations should stack on tablet
