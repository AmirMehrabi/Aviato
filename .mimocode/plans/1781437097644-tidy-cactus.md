# Customer Index Page Redesign + Scroll Fix

## Problem
1. The dark navy hero banner (`bg-[#031B4E]`) takes too much space with no useful information
2. When searching/filtering, the page scrolls to the top due to full-page form submission

## Approach

### 1. Replace dark header with simple flex header
Replace the 20-line dark banner (lines 18-37) with the same pattern used by `virtual-machines/index`, `tickets/index`, `cloud-images/index` — a simple flex row:

```blade
<div class="flex flex-col gap-4 md:flex-row md:items-center md:justify-between">
    <div>
        <h1 class="text-2xl font-black">مشتریان</h1>
        <p class="mt-2 text-sm text-slate-500">مدیریت مشتریان و وضعیت سرویس‌ها</p>
    </div>
    <a href="{{ route('admin.customers.create') }}" class="rounded-lg bg-[#0069FF] px-5 py-3 text-sm font-black text-white">افزودن مشتری</a>
</div>
```

- Remove the stats cards grid entirely (they can be seen on the dashboard or inferred from the table)
- Keep the `$stats` data in the controller so it's available if needed later

### 2. Fix scroll-to-top on search
The root cause: the form does a full GET page reload, which resets scroll to top.

**Solution:** Convert the search/filter form to AJAX-based filtering using Alpine.js.

- Wrap the table in a container `<div x-ref="results">`
- Intercept form submission with Alpine.js: prevent default, serialize form data, fetch the same URL with `?` + query string via `fetch()`, parse the HTML response, extract the table content, swap it in
- Update the browser URL with `history.pushState` so the URL stays in sync
- No scroll jump because the DOM updates in-place without a page reload

The Alpine.js component will be updated to:
```javascript
x-data="{
    timer: null,
    submit() { clearTimeout(this.timer); this.timer = setTimeout(() => this.fetchResults(), 450); },
    fetchResults() {
        const params = new FormData(this.$refs.filters);
        const url = this.$refs.filters.action + '?' + new URLSearchParams(params).toString();
        history.pushState({}, '', url);
        fetch(url, { headers: { 'X-Requested-With': 'XMLHttpRequest' } })
            .then(r => r.text())
            .then(html => {
                const doc = new DOMParser().parseFromString(html, 'text/html');
                this.$refs.results.innerHTML = doc.querySelector('[x-ref=\"results\"]').innerHTML;
            });
    }
}"
```

- Change `submit()` calls to `fetchResults()` on all filter inputs
- Change `<form>` to `<form @submit.prevent>` to prevent default submission
- Add `x-ref="results"` wrapper around the table section
