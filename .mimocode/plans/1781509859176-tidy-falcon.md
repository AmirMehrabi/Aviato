# Plan: Add Customer Display Name + OS-Based Internal Naming

## Context

Currently `VirtualMachine.name` serves double duty — it's the auto-generated Proxmox identifier (`MACHINE-YYMM-{specs}-{random}`) AND what customers see in the dashboard. This is non-standard: every major VPS provider (DigitalOcean, Hetzner, Linode, Vultr) separates the internal identifier from a customer-facing display name.

**Goal:**
1. Change the internal auto-generated name from `MACHINE-YYMM-{specs}-{random}` to `{OS}-{YYMM}-{specs}-{random}` (e.g. `UBNT-2606-2C4G20G-ABCDEF`).
2. Add a new `display_name` column that customers set at creation and can edit anytime.
3. Show `display_name` in customer panels; show both in admin panels.

## Naming Convention

| `os_family` | Prefix | Example |
|---|---|---|
| `ubuntu` | `UBNT` | `UBNT-2606-2C4G20G-ABCDEF` |
| `debian` | `DBN` | `DBN-2606-2C4G20G-XYZ789` |
| `rocky` | `RKL` | `RKL-2606-2C4G20G-PQRST` |
| `router_os` | `ROS` | `ROS-2606-2C4G20G-123456` |
| `windows` | `WND` | `WND-2606-2C4G20G-UVWXYZ` |

Format: `{OS_PREFIX}-{YYMM}-{CPU}C{RAM}G{DISK}G-{RANDOM6}`
